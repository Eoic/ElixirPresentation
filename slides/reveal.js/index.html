The Elixir Programming Language

Erlang

  • A development platform for building scalable systems.
  • Created at Ericsson in 1980s.
  • Developed for handling telephone switches with millions of concurrent connections.
  • Banned by Ericsson for internal use in 1998 (open sourced the same year).

Erlang goals

  • Distribution.
  • Fault tolerance.
  • High availability.
  • Responsiveness.
  • Live updates.

Erlang concurrency

Elixir

  • Created as a R&D project at Platformatec (closed) by Jos√© Valim.
  • First release in 2012.
  • An alternative language for the Erlang virtual machine.
  • Makes Erlang platform more approachable.
  • Simplifies syntax, reduces boilerplate and duplication.
  • Compiles into BEAM-compliant bytecode
  • No performance penalty.
  • Extensible through macros.

Server process in Erlang 

							
								-module(sum_server).
								-behaviour(gen_server).

								-export([
									start/0,
									sum/3,
									init/1,
									handle_call/3,
									handle_cast/2,
									handle_info/2,
									terminate/2,
									code_change/3
								]).

								start() -> gen_server:start(?MODULE, [], []).
								sum(Server, A, B) -> gen_server:call(Server, {sum, A, B}).

								init(_) -> {ok, undefined}.
								handle_call({sum, A, B}, _From, State) -> {reply, A + B, State}.
								handle_cast(_Msg, State) -> {noreply, State}.
								handle_info(_Info, State) -> {noreply, State}.
								terminate(_Reason, _State) -> ok.
								code_change(_OldVsn, State, _Extra) -> {ok, State}.

Server process in Elixir 

							
								defmodule SumServer do
									use GenServer

									def start do
										GenServer.start(__MODULE__, nil)
									end

									def sum(server, a, b) do
										GenServer.call(server, {:sum, a, b})
									end

									def handle_call({:sum, a, b}, _from, state) do
										{:reply, a + b, state}
									end
								end
